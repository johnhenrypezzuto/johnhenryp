Open science practices are on the rise. However I don’t really think good coding practices are there yet. There are a few really good guides on good workflow practices that I found especially helpful, but I didn’t really find these modern or comprehensive (or entirely focused in R!). This is my attempt at such a guide.

Before going further I do want to emphasize that it makes sense to post data and code to a place like OSF. Academics move institutions which causes links, people forget to renew personal websites. A place like OSF has their reputation on the line to keep these links absolute. Make it their problem not yours. Uri Simonsohn also has a guide specifically about keeping website links alive for those interested.

Managing References

When creating a dynamic document it makes sense to use a reference manager such as Zotero. Zotero is pretty easy to use and I won’t get into it, but one thing I initially had trouble with was using it to make BibTex files.

The problem is that Zotero defaults the citation key as “first author last name, title, year”, which is just way too long to use on the fly. To make Zotero use shorter citation keys (such as “first author last name, year”) Here is how you do it:

  1. Download Better BibTeX for Zotero

  2. In the Zotero settings (⌘ I) make sure the citation key is set to “[auth:lower][year]”

  1. You need to update all of your previous references now. The way you do this is by selecting all (⌘ A) right clicking, and going down to the Better BibTeX menu.
